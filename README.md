## рҹ”Қ Overview

This tool was developed for **ANPER (Agence NigГ©rienne pour la
Promotion de l'Гүlectrification Rurale)** to support decision-makers
in identifying and prioritizing rural electrification sites in Niger
using the VIDA dataset.

The app allows stakeholders to:
- Explore **26,000+ settlement sites** on an interactive map
- Filter sites by **region, department, energy demand, security
  risk, population, and viability score**
- Visualize **individual site details** via clickable popups
- Analyze data through **KPI dashboards**
- Download **filtered datasets** as CSV

## вңЁ Features

### рҹ—әпёҸ Interactive WebMap
- Real **OpenStreetMap / Satellite / Terrain / Dark** basemaps
- **MarkerCluster** for smooth navigation at any zoom level
- **Individual popups** for every site (name, score, population,

## вҡҷпёҸ Installation

### 1) Clone or download the project
```bash
git clone https://github.com/your-username/anper-vida-webmap.git
cd anper-vida-webmap
```

### 2) Create a virtual environment (recommended)
```bash
# With conda
conda create -n streamlit_env python=3.11
conda activate streamlit_env

# Or with venv
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Linux/Mac
```

### 3) Install dependencies
```bash
pip install -r requirements.txt
```

### 4) Place your data file
Put the original Excel file in the `data/` folder:
```
data/Niger VIDA tool.xlsx
```

### 5) Run the preprocessing script (once)
```bash
python prepare_vida_data.py
```

This generates:
```
data/vida_clean.parquet
```

### 6) Launch the app
```bash
streamlit run app.py
```

Open your browser at: **http://localhost:8501**

## рҹ”„ Data Pipeline (Plan B)

This app uses a **two-step data pipeline** to ensure fast loading
and stable column names, avoiding issues with the multi-row
merged-cell headers in the original Excel file.

```
Niger VIDA tool.xlsx
        в”Ӯ
        в–ј
prepare_vida_data.py
  вҖў Auto-detects header row
  вҖў Removes Unnamed columns
  вҖў Casts numeric columns
  вҖў Saves clean parquet
        в”Ӯ
        в–ј
data/vida_clean.parquet
        в”Ӯ
        в–ј
utils/data_loader.py (cached)
  вҖў Loads parquet instantly
  вҖў Standardizes risk labels
  вҖў Computes viability score
  вҖў Adds class labels
        в”Ӯ
        в–ј
Streamlit App (filters вҶ’ map вҶ’ KPIs)
```

**Why Parquet?**
- Loads ~10x faster than Excel
- Preserves correct data types
- No header detection needed at runtime
- Smaller file size

## рҹ—әпёҸ Viability Score Methodology

The viability score (0вҖ“100) is computed automatically from
four normalized indicators:

```
Score = 0.30 Г— norm(Connections)
      + 0.25 Г— norm(Energy Demand)
      + 0.25 Г— norm(PV Potential)
      + 0.20 Г— norm(Wealth Index)
```

Each indicator is **min-max normalized** across all sites.
Missing values are replaced by the column median before
normalization to avoid penalizing sites with incomplete data.

| Score Range | Class |
|-------------|-------|
| 75 вҖ“ 100 | рҹҹў Excellente |
| 50 вҖ“ 75 | рҹҹЎ Bonne |
| 25 вҖ“ 50 | рҹҹ  ModГ©rГ©e |
| 0 вҖ“ 25 | рҹ”ҙ Faible |
