## 🔍 Overview

This tool was developed for **ANPER (Agence Nigérienne pour la
Promotion de l'Électrification Rurale)** to support decision-makers
in identifying and prioritizing rural electrification sites in Niger
using the VIDA dataset.

The app allows stakeholders to:
- Explore **26,000+ settlement sites** on an interactive map
- Filter sites by **region, department, energy demand, security
  risk, population, and viability score**
- Visualize **individual site details** via clickable popups
- Analyze data through **KPI dashboards**
- Download **filtered datasets** as CSV

## ✨ Features

### 🗺️ Interactive WebMap
- Real **OpenStreetMap / Satellite / Terrain / Dark** basemaps
- **MarkerCluster** for smooth navigation at any zoom level
- **Individual popups** for every site (name, score, population,
  demand, risk, PV potential, etc.)
- **Color-coded markers** by:
  - Viability Score
  - Security Risk (Low / Medium / High)
  - Population
  - Energy Demand
  - Night-light intensity
  - Wealth Index
- **Size-coded markers** by population, connections, or demand
- **Legend** overlay on the map
- **Fullscreen** and **MiniMap** controls

### 📊 KPI Dashboard
Live metrics updated with every filter:

| KPI | Description |
|-----|-------------|
| 🏘️ Sites | Number of filtered sites |
| 👥 Population | Total population covered |
| ⚡ Avg Demand | Average energy demand (kWh/day) |
| 🔌 Avg Connections | Average estimated connections |
| 🏆 Avg Score | Average viability score (0–100) |
| ⚠️ High Risk % | % of sites with High security risk |

### 🔍 Smart Filters (Apply Button)
Filters only apply when you click **✅ Apply Filters**,
preventing unnecessary reruns and keeping the app responsive:

- 📍 Region & Department
- 🏆 Viability Score range
- ⚡ Energy demand range
- 🔌 Estimated connections range
- 📏 Distance to existing grid
- 🛡️ Security risk level (Low / Medium / High)
- ⚙️ Max points to draw on map (performance)
- 🖱️ Enable/disable popups

### 📋 Data Table
- View filtered data with key columns
- Sort by viability score
- **Download filtered dataset as CSV**

## ⚙️ Installation

### 1) Clone or download the project
```bash
git clone https://github.com/your-username/anper-vida-webmap.git
cd anper-vida-webmap
```

### 2) Create a virtual environment (recommended)
```bash
# With conda
conda create -n streamlit_env python=3.11
conda activate streamlit_env

# Or with venv
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Linux/Mac
```

### 3) Install dependencies
```bash
pip install -r requirements.txt
```

### 4) Place your data file
Put the original Excel file in the `data/` folder:
```
data/Niger VIDA tool.xlsx
```

### 5) Run the preprocessing script (once)
```bash
python prepare_vida_data.py
```

This generates:
```
data/vida_clean.parquet
```

### 6) Launch the app
```bash
streamlit run app.py
```

Open your browser at: **http://localhost:8501**

## 🚀 Usage

### First time setup
```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Preprocess data (one time only)
python prepare_vida_data.py

# 3. Run the app
streamlit run app.py
```

### When the Excel file is updated
```bash
# Re-run preprocessing to refresh the parquet
python prepare_vida_data.py

# Restart the app
streamlit run app.py
```

### Using the app
1. **Adjust filters** in the left sidebar
2. Click **✅ Apply Filters** to update the map and KPIs
3. **Click any marker** on the map to see site details
4. Switch tabs between **🗺️ Map** and **📋 Data**
5. **Download** the filtered dataset from the Data tab

## 🔄 Data Pipeline (Plan B)

This app uses a **two-step data pipeline** to ensure fast loading
and stable column names, avoiding issues with the multi-row
merged-cell headers in the original Excel file.

```
Niger VIDA tool.xlsx
        │
        ▼
prepare_vida_data.py
  • Auto-detects header row
  • Removes Unnamed columns
  • Casts numeric columns
  • Saves clean parquet
        │
        ▼
data/vida_clean.parquet
        │
        ▼
utils/data_loader.py (cached)
  • Loads parquet instantly
  • Standardizes risk labels
  • Computes viability score
  • Adds class labels
        │
        ▼
Streamlit App (filters → map → KPIs)
```

**Why Parquet?**
- Loads ~10x faster than Excel
- Preserves correct data types
- No header detection needed at runtime
- Smaller file size

## ⚙️ Configuration

All key settings are in `config.py`:

```python
# Map center (Niger)
MAP_CENTER = [17.6078, 8.0817]
MAP_ZOOM   = 6

# Column name mapping
COL = {
    "lat"         : "Latitude",
    "lon"         : "Longitude",
    "name"        : "Nom",
    "region"      : "Région",
    "dept"        : "Départment",
    "connections" : "Nombre estimé de connexions",
    "demand"      : "Demande énergétique estimée [kWh/day]",
    "risk"        : "Risque de sécurité",
    ...
}

# Viability score weights (must sum to 1.0)
VIABILITY_WEIGHTS = {
    "connections" : 0.30,
    "demand"      : 0.25,
    "pv"          : 0.25,
    "wealth"      : 0.20,
}

# Security risk colors
RISK_COLORS = {
    "Low"    : "#2ecc71",
    "Medium" : "#f39c12",
    "High"   : "#e74c3c",
}
```

To adapt this app to another country's VIDA dataset:
1. Update `MAP_CENTER` and `MAP_ZOOM`
2. Update `COL` mapping if column names differ
3. Replace the Excel file in `data/`
4. Re-run `prepare_vida_data.py`

## 🗺️ Viability Score Methodology

The viability score (0–100) is computed automatically from
four normalized indicators:

```
Score = 0.30 × norm(Connections)
      + 0.25 × norm(Energy Demand)
      + 0.25 × norm(PV Potential)
      + 0.20 × norm(Wealth Index)
```

Each indicator is **min-max normalized** across all sites.
Missing values are replaced by the column median before
normalization to avoid penalizing sites with incomplete data.

| Score Range | Class |
|-------------|-------|
| 75 – 100 | 🟢 Excellente |
| 50 – 75 | 🟡 Bonne |
| 25 – 50 | 🟠 Modérée |
| 0 – 25 | 🔴 Faible |
